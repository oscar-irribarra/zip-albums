# Implementation Plan: Image Zoom Fix

**Branch**: `013-image-zoom-fix` | **Date**: 2026-07-11 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/013-image-zoom-fix/spec.md`

## Summary

A zoom system already exists in `ViewerScreen.tsx` but was implemented with incorrect constants and behavioral divergences from the specification. This plan corrects those bugs:

- `ZOOM_STEP` 0.25 → 0.10 (10% per activation)
- `ZOOM_MIN` 0.5 → 0.25 (25% minimum scale)
- Pan enabled at all zoom levels, not just above 100%
- Pan is unconstrained — no boundary clamping

No backend, filesystem, or data-persistence changes are required. The entire fix is contained in a single frontend component.

## 1. Feature Overview

The image viewer in this application already renders a zoom and pan UI, but the implementation diverges from specification in four concrete ways:

1. The zoom step is 0.25 (25%) instead of 0.10 (10%).
2. The minimum scale guard in the component is 0.5 (50%) instead of 0.25 (25%).
3. Drag/pan is disabled at 100% scale but the spec allows it at any scale.
4. Pan is constrained to image bounds; the spec requires unconstrained pan.

This feature corrects these defects. The result is a viewer where zoom steps are 10%, the scale range is 25%–400%, panning works at any zoom level, and the image can be freely dragged in any direction.

## 3. Technical Architecture

This is a pure frontend fix. The responsibility boundary is clear:

```
┌────────────────────────────────────────────────────────────┐
│  FRONTEND (React / TypeScript)                             │
│                                                            │
│  ViewerScreen.tsx                                          │
│  ├─ Constants: ZOOM_STEP, ZOOM_MIN, ZOOM_MAX               │
│  ├─ Zoom handlers: handleZoomIn, handleZoomOut,            │
│  │                 handleZoomReset                         │
│  ├─ Pan handlers: handlePointerDown, handlePointerMove,    │
│  │                handlePointerUp                          │
│  └─ Render: <img style="transform: translate+scale" />     │
│                                                            │
│  libraryStore.ts (Zustand)                                 │
│  ├─ state: zoomLevel (number, persisted in memory only)    │
│  ├─ action: setZoomLevel → clampZoomLevel [0.25, 4.0]      │
│  └─ navigation actions reset zoomLevel to 1               │
└────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────┐
│  BACKEND (Rust / Tauri)                                    │
│  No changes. Zoom/pan do not involve any Tauri commands.   │
└────────────────────────────────────────────────────────────┘
```

**Frontend responsibilities**:
- Maintain `scale` and `panOffset` state
- Enforce ZOOM_STEP, ZOOM_MIN, ZOOM_MAX constants
- Apply `transform: translate(x, y) scale(s)` to the image element
- Handle pointer events for drag interaction
- Reset zoom and pan on navigation

**Backend (Rust) responsibilities**: None for this feature.

**Shared models**: None — zoom state is ephemeral and frontend-only.

**Infrastructure services**: None — no filesystem access, no ZIP reads.

## 8. React Components

### `ViewerScreen.tsx` — diff summary

**Fix 1: Constants**
```typescript
// Before
const ZOOM_STEP = 0.25;
const ZOOM_MIN  = 0.5;

// After
const ZOOM_STEP = 0.10;
const ZOOM_MIN  = 0.25;
```

**Fix 2: Remove drag gate**
```typescript
// Before
const handlePointerDown = (e: React.PointerEvent<HTMLDivElement>) => {
  if (zoomLevel <= 1) return;   // ← remove this line
  ...
};

// After
const handlePointerDown = (e: React.PointerEvent<HTMLDivElement>) => {
  isDragging.current = true;
  ...
};
```

**Fix 3: Remove pan boundary clamping**
```typescript
// Before (handlePointerMove)
const frame = frameRef.current.getBoundingClientRect();
const maxX = Math.max(0, (prevImageSize.width * zoomLevel - frame.width) / 2);
const maxY = Math.max(0, (prevImageSize.height * zoomLevel - frame.height) / 2);
const newX = dragStart.current.panX + (e.clientX - dragStart.current.x);
const newY = dragStart.current.panY + (e.clientY - dragStart.current.y);
setPanOffset({
  x: Math.max(-maxX, Math.min(newX, maxX)),
  y: Math.max(-maxY, Math.min(newY, maxY)),
});

// After (handlePointerMove)
const newX = dragStart.current.panX + (e.clientX - dragStart.current.x);
const newY = dragStart.current.panY + (e.clientY - dragStart.current.y);
setPanOffset({ x: newX, y: newY });
```

Also remove the `prevImageSize` guard from `handlePointerMove` since the frame bounds check is no longer needed:

```typescript
// Before
const handlePointerMove = (e: React.PointerEvent<HTMLDivElement>) => {
  if (!isDragging.current || !frameRef.current || !prevImageSize) return;

// After
const handlePointerMove = (e: React.PointerEvent<HTMLDivElement>) => {
  if (!isDragging.current) return;
```

**Fix 4: CSS class condition**
```tsx
// Before
className={`album-viewer-image-frame${zoomLevel > 1 ? " album-viewer-image-frame--zoomed" : ""}`}

// After
className="album-viewer-image-frame album-viewer-image-frame--zoomed"
```

The `frameRef` is still needed for nothing in the new implementation; however it is harmless to keep. Remove the `frameRef` ref if no other usage exists in the final code.

> **Note**: `prevImageSize` state is still required for the skeleton aspect-ratio calculation (`skeletonStyle`). Do not remove it — only remove its use in `handlePointerMove`.

## 10. Error Handling

**No new error handling required.**

- Pan offset is unconstrained integers; no validation needed.

## 12. Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Unconstrained pan: image dragged fully off screen | Low | Low | Spec explicitly requires unconstrained pan; user can always click Reset |
| `prevImageSize` removal causes skeleton aspect ratio to break | Low | Medium | `prevImageSize` is **not removed** — only its use in the pan handler is removed |
| Floating-point accumulation at scale boundaries | Low | Low | `clampZoomLevel` in the store catches any overshoot; rounding errors ≪ 0.01 |

## 13. Future Extensibility

The following are **out of scope** for this feature but can be added later without architectural changes:

- **Scroll-wheel zoom**: Add a `wheel` event handler to `ViewerScreen.tsx` calling `handleZoomIn`/`handleZoomOut`.
- **Pinch-to-zoom**: Add pointer event handling for multi-touch gestures in `ViewerScreen.tsx`.
- **Zoom percentage display**: Add a `<span>` showing `Math.round(zoomLevel * 100)%` next to the zoom controls.
- **Soft-boundary pan (spring-back)**: Replace unconstrained pan with a pan that springs back when the image is dragged beyond its bounds. Requires adding bounds calculation back to `handlePointerMove` and a CSS transition.
- **Keyboard shortcuts for zoom**: Add `+`/`-`/`0` key bindings to the existing `handleKeyDown` handler in `ViewerScreen.tsx`.
- **Zoom to cursor position**: Transform origin tied to pointer position rather than center; requires adjusting `transformOrigin` dynamically.
