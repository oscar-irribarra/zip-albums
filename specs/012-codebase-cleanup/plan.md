## Testing Strategy

### Regression baseline

Run `pnpm test` and `cargo test` before making any change. All tests must pass at the
start to establish a clean baseline.

### Per-change validation

After each deletion or update, re-run `pnpm test` to verify no regression was introduced.
Do not batch multiple changes before running tests.

### After all changes

Run `pnpm build` to confirm the TypeScript compiler reports no errors against the
post-cleanup source tree. Run `cargo build` with warnings to confirm no Rust dead-code
warnings remain.

### Test coverage impact

- `ImageViewer.test.tsx` is deleted. `ViewerScreen` behavior is covered by
  `libraryStore.test.ts` (viewer navigation, cache, progress) and the smoke test.
- `src/test/libraryStore.test.ts` is deleted. Its coverage is fully provided by
  `src/features/library/store/libraryStore.test.ts`.

No net reduction in coverage of live code paths.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| `loadAlbumImageForCache` is mocked by name in a test not yet found | Low | Test breaks | Grep all test files for the string `loadAlbumImageForCache` before removing |
| `ImageViewer` is imported via a dynamic path or string lookup | Very low | Runtime error | Grep for `"ImageViewer"` and `'ImageViewer'` as strings; confirm zero hits |
| Removing `greet` breaks a CI step that calls it | Very low | Build failure | Confirm no pipeline script invokes `greet`; the Tauri dev toolchain has no test that calls `greet` |
| `ShortcutGesture` / `ShortcutGuardContext` have usage added after this research | Low | TypeScript error | Run `pnpm build` after deletion; the compiler will surface any new usage immediately |
